# Приложение 2. Сервисные возможности и администрирование

В программе реализован широкий набор сервисных возможностей, упрощающих работу пользователей и администраторов.

## 2.1. Права доступа

### Роли пользователей

Программа поставляется с предопределенными ролями:
- **Администратор**
- **Кадровик**
- **Расчетчик зарплаты**
- **Кадровик-расчетчик**
- **Руководитель подразделения**
- **Табельщик**
- **Сотрудник** (самообслуживание)

Роли можно комбинировать.

### Ограничение доступа по объектам

Можно настроить доступ:
- По организациям
- По подразделениям
- По группам доступа

Для этого используются:
- **Группы доступа**
- **Профили групп доступа**
- **Ограничение доступа на уровне записей** (RLS)

### Дополнительные настройки прав

- **Использование дополнительных реквизитов и сведений**
- **Редактирование реквизитов объектов**
- **Просмотр истории изменений**

## 2.2. Дополнительные реквизиты и сведения

Можно добавлять собственные реквизиты к любым объектам программы (справочникам, документам, планам видов характеристик и т.д.) без изменения конфигурации.

**Примеры использования**:
- Дополнительные характеристики должностей
- Сведения о семейном положении (если не хватает стандартных)
- Признаки для специфических расчетов

## 2.3. История изменений объектов

Для большинства справочников и документов можно включить **хранение истории изменений**.

**Возможности**:
- Просмотр кто, когда и что изменил
- Сравнение версий объекта
- Восстановление предыдущей версии (в некоторых случаях)

## 2.4. Журнал регистрации

**Журнал регистрации** фиксирует все значимые события в информационной базе:
- Вход/выход пользователей
- Создание, изменение, удаление объектов
- Проведение/отмена проведения документов
- Ошибки и предупреждения

**Использование**:
- Поиск действий конкретного пользователя
- Анализ проблем
- Аудит безопасности

## 2.5. Резервное копирование

### Настройка резервного копирования

В разделе **Администрирование → Обслуживание** можно настроить:
- Автоматическое создание резервных копий
- Периодичность
- Количество хранимых копий
- Каталог для хранения

### Ручное создание копии

Команда **Администрирование → Обслуживание → Резервное копирование и восстановление → Создать резервную копию**.

### Восстановление из копии

Выполняется через **Конфигуратор** или через режим **1С:Предприятие** (при наличии соответствующих прав).

## 2.6. Обновление программы

### Обновление через интернет

Программа может автоматически проверять наличие обновлений и скачивать их.

### Ручное обновление

1. Скачать файл обновления с сайта 1С или портала ИТС
2. Запустить обновление из режима **1С:Предприятие** или через **Конфигуратор**

**Важно**: Перед обновлением рекомендуется сделать резервную копию.

## 2.7. Монитор производительности и диагностика

- **Журнал регистрации** (см. выше)
- **Технологический журнал** (для сложной диагностики)
- **Центр управления производительностью** (в расширенной версии)

## 2.8. Работа с файлами и вложениями

Многие объекты программы поддерживают прикрепление файлов (сканы документов, фотографии сотрудников, договоры и т.д.).

**Возможности**:
- Прикрепление файлов к карточкам сотрудников, документам
- Хранение файлов в информационной базе или во внешнем каталоге
- Контроль размера и типа файлов

## 2.9. Настройка списка пользователей и аутентификация

- Создание и редактирование пользователей
- Назначение ролей
- Настройка аутентификации (1С, Windows, OpenID)
- Ограничение по IP-адресам (при необходимости)
- Настройка паролей и политики их сложности

## 2.10. Другие сервисные возможности

- **Поиск и замена ссылок** (при удалении дублей)
- **Удаление помеченных объектов**
- **Проверка и исправление данных**
- **Групповая обработка справочников и документов**
- **Консоль запросов** (для разработчиков)
- **Консоль управления системой** (в клиент-серверном варианте)

---

**Рекомендация**: Администратору информационной базы рекомендуется регулярно проверять **Журнал регистрации**, настраивать автоматическое резервное копирование и контролировать права доступа пользователей.
